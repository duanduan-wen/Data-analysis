# 库存与销售数据分析项目
## 项目背景
针对多仓库、多商品的库存管理场景，结合销售数据完成**库存状态监控、销售趋势分析、库存健康度评估**，核心解决“临期库存预警、库存可用天数计算、销售ABC分析”等业务问题，为库存优化提供数据支撑。

## 技术栈
- 数据处理：Pandas、NumPy（数据清洗、聚合、指标计算）
- 可视化：Plotly（交互式图表，支持下拉筛选、多维度联动）
- 工程化：模块化设计、路径管理、异常处理

## 核心功能模块
### 1. 数据预处理模块
- 库存数据：多文件合并、仓库补全（含0库存）、状态标记（过期/临期/常规）、库存类型标记（在途/采购/库存）
- 销售数据：列名清洗、时间筛选、商品过滤、区域-仓库映射
- 指标计算：日均销量、库存可用天数、标准天数对比、累计占比（帕累托）

### 2. 可视化模块（交互式图表）
| 图表名称 | 业务价值 |
|---------|---------|
| 库存状态饼图 | 按商品查看过期/临期/常规库存分布，关联仓库信息 |
| 仓库标记堆叠柱图 | 按商品查看各仓库库存类型（在途/采购/库存）占比 |
| 销售趋势折线图 | 查看商品日销量趋势，支持圆滑曲线+数值标注 |
| 销售帕累托图 | ABC分析，识别核心销售商品 |
| 商品可用天数对比图 | 库存健康度评估（实际可用天数 vs 标准天数） |
| 仓库可用天数对比图 | 分仓库查看库存健康度，支持商品下拉筛选 |
